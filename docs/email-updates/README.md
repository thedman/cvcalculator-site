# CVCalculator Email Update Scaffold

## Purpose

This folder supports future low-frequency educational email updates for people who have subscribed to CVCalculator rate and article updates.

The intended use is to help subscribers understand material changes related to commuted values, pension transfer limits, tax treatment, and rate movement. The emails should support education and context, not replace professional financial, tax, legal, or actuarial advice.

## Current Status

This is a scaffold only.

No sending system exists in this repository. There is no email service integration, automation, cron job, subscriber database, API workflow, delivery configuration, or production email process.

## Intended Audience

Canadian defined-benefit pension members researching:

- Commuted values
- LIRA transfer limits
- Tax treatment of taxable commuted value amounts
- CIA Section 3500 rate movement
- Deferred pension versus commuted value decisions
- Employer departure and pension option timelines

## Appropriate Send Triggers

Future updates should be sent only when there is a clear educational reason. Appropriate triggers include:

- CIA long-term rate update
- Material pension transfer or tax article update
- Significant new educational guide

## Non-Goals

These email updates should not become:

- Personalized financial advice
- Personalized tax, legal, actuarial, or pension advice
- Automated trading or investment commentary
- A high-frequency newsletter
- A market-timing alert system
- An email platform implementation
- A substitute for official pension plan communications

No email platform should be implemented from this scaffold alone.

## Tone Principles

Use a tone that is:

- Educational
- Cautious
- Transparent
- Plain-language
- Specific about what changed and what did not

The CVCalculator app CTA should be secondary. The primary value of each email should be helping the reader understand the pension context.

## Compliance Reminder

Do not send real emails until consent, unsubscribe handling, privacy language, sender identity, delivery mechanics, and recordkeeping have been confirmed.
